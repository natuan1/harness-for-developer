# Constitution — [Project Name]

These are immutable principles for this project. All code, specs, and decisions must comply.

## Article I: Spec-First Development

Every feature begins with a specification. Code is the expression of that specification.
No implementation without an approved spec.

## Article II: Test-First Imperative

Write tests before implementation code. No exceptions.
Tests define what "done" means before we start building.

## Article III: One Feature at a Time

Work on exactly one feature at a time. No parallel features.
Complete, verify, commit — then move to the next.

## Article IV: No Premature Abstraction

Three similar patterns > create an abstraction.
Don't design for hypothetical future requirements.
The right complexity is what the task actually requires.

## Article V: Verify Before Declare

Never mark work as done without running verification.
Evidence required: test output, type-check results, self-review notes.

## Article VI: Document Decisions

Every technical decision must be recorded in an ADR or progress log.
Future developers (including AI agents) need to know WHY, not just WHAT.

## Article VII: Human-in-the-Loop at Gates

Phase transitions require human approval.
The implementation readiness gate is not optional.
Trust the agent between gates; verify at gates.
